Feature 004 "advisor exploratory phase" adds a pre-findings step where each participating advisor builds + persists a lens understanding, with an orchestrator-run batched operator interview per round.

Product-lens bedrock (the through-line of my PRIORITIZE votes at GATE A): **the operator pays — in time, attention, and unintended writes — for a phase nobody has yet proven the operator needs.**

Concrete findings I escalated/affirmed:
- Built ahead of proof that missing understanding is the binding constraint on review quality (if it isn't, the phase serves roster rigor, not the operator).
- Standing per-round tax (interview + per-lens analysis + upkeep) is unpriced recurring excise.
- No cheaper viable subset sequenced first → full tax paid before it's shown to buy anything; wrong default.
- "Bounded analysis" with no defined budget = cost the operator can't forecast.
- One batched interview serializes the whole panel behind a single operator gate (blocking excise).
- **Rubber-stamp trap**: FR-017 says project-wide facts are written to the operator-owned addendum (CHORUS-PROJECT.md) only as "proposed additions the operator accepts, never auto-committed." Under the per-round fatigue the tax creates, "accept" becomes reflexive → the "never auto-committed" guarantee becomes fiction. A consent control defeated by fatigue is no control. (Links to [[feedback-consent-encounter-not-display]] — consent is an encounter, not a display; reflexive accept is display.)
- One shared tag for lens-specific vs project-wide facts lets a lens-specific inference ride into the shared addendum the operator owns — worsens the rubber-stamp.

**Why:** these compound silently across rounds; per-round cost-shifts are exactly the kind that become invisible.
**How to apply:** if 004 proceeds, demand (a) proof understanding-gaps degrade reviews before charging the operator every round, (b) a priced per-round budget, (c) a write-back control stronger than blanket "accept" — per-field encounter, not a single stamp.

**Gate A cycle 2 (2026-06-07) — CLEARED on my lens.** Spec revised: FR-019 (sessioned ≤5-question, re-entrant defer/resume, operator-paced budget, educational preamble), FR-020 (cheapest-subset-first), SC-009 + gap-interview.md degradation summary. The two gating 🔴 I raised resolved:
- **F8 (unbounded interview)** RESOLVED — per-session cap + operator pacing + cheapest-subset-first. Per-session cap (not hard total) is the *honest* design: a hard total would silently drop real gaps; pacing lets the operator decide when value-to-cost stops paying. Checked the lurking worry (re-entrant → never-ending interview): structurally bounded — question pool is finite genuine gaps (reference-harvest + bounded-analysis + dedupe + FR-008 run first); sessions partition a finite pool, never generate questions; defer always has a safe degraded outcome.
- **F10 (silent degraded verdict)** RESOLVED — verdict-level degradation summary (gaps remaining + findings affected), open gaps fail safe as provisional.
Standing-tax bedrock is now bounded + operator-controlled, NOT open. Residual non-gating nit: educational preamble repeats every session — excise for the returning operator; suggested first-full / later-one-line reminder. Author's call, not a halt.
